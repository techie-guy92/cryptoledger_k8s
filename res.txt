## Question 1: Why No Port `8003` in URL?

**Because Ingress listens on port 80 (default HTTP port).**

| Access method | Port | Why |
|---------------|------|-----|
| **Ingress** | `http://crypto.local/admin/` | Ingress controller listens on port 80 |
| **Service (NodePort)** | `http://crypto.local:32708/admin/` | Direct service access requires the node port |

**Ingress acts as a reverse proxy** — it receives traffic on port 80, then forwards it internally to your service on port 8003.

```
Browser → Port 80 → Ingress Controller (Traefik) → Service (ClusterIP:8003) → Pod (8003)
```

So you don't need to specify `:8003` because Ingress handles the routing.

## Question 2: Server Error (500) After Login

This is likely due to **missing database tables** or **session/cookie configuration**. Let's debug.

---

### Step 1: Check Django Logs

```bash
kubectl logs -n cryptoledger -l app=cryptoledger --tail=50
```

Look for the specific error when you try to log in.

---

### Step 2: Common Causes of 500 After Login

| Cause | Solution |
|-------|----------|
| **Missing `django_session` table** | Run migrations again |
| **CSRF token issue** | Check `CSRF_TRUSTED_ORIGINS` |
| **Session backend misconfigured** | Check `SESSION_ENGINE` |
| **Database connection error** | Check if Postgres is reachable |

---

### Step 3: Run Migrations Again

```bash
kubectl exec -it -n cryptoledger deployment/cryptoledger-web -- python manage.py migrate
```

---

### Step 4: Check if Session Table Exists

```bash
kubectl exec -it -n cryptoledger deployment/cryptoledger-web -- python manage.py dbshell -c "\dt"
```

Or:

```bash
kubectl exec -it -n cryptoledger deployment/cryptoledger-web -- python -c "
from django.db import connection
tables = connection.introspection.table_names()
print('django_session exists:', 'django_session' in tables)
"
```

### Step 5: Check CSRF_TRUSTED_ORIGINS

Your ConfigMap has:

```
CSRF_TRUSTED_ORIGINS: "https://crypto.local:7443,..."
```

But you're accessing via `http://crypto.local` (not HTTPS, not port 7443). Add:

```
http://crypto.local
```

Update ConfigMap:

```bash
kubectl edit configmap -n cryptoledger cryptoledger-config
```

Add to `CSRF_TRUSTED_ORIGINS`:

```
http://crypto.local,http://crypto.local:80
```

Then restart deployment:

```bash
kubectl rollout restart deployment -n cryptoledger cryptoledger-web
```
